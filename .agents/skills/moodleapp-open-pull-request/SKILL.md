# Moodle Mobile App Open Pull Request

This skill guides you through creating a pull request to contribute your changes to the open source Moodle Mobile App repository.

## When to Use This Skill

- You have made changes to the Moodle Mobile App codebase and want to contribute them back to the community.

## Steps to Create a Pull Request

1. If enabled, using the Atlassian MCP server get the issue title and description. Use the actual branch name as the issue key.
2. If needed, push the changes on your local branch to your forked repository on GitHub.
3. Using github MCP server, create a pull request from your branch to the main branch of the Moodle Mobile App repository.
4. Include the issue title and description in the pull request to provide context for reviewers.
5. Submit the pull request.
